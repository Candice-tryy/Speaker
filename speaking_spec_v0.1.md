# Speaking — 雅思口语练习 App 产品规格 v0.1

> 一款面向中国雅思考生的口语练习工具，以「语料库卡片练习 + 游戏化登山闯关」为核心。
> 平台优先级：先做 Web 版验证玩法 → iOS（也可能微信小程序）。
> 本文件为产品规格（PRD + 设计稿说明），供开发与迭代使用。

---

## 1. 产品定位

- **一句话**：用「卡片练习 + 登山闯关」把枯燥的雅思口语语料背诵游戏化，让考生敢开口、练得下去、看得见进步。
- **核心差异点**：不是又一个题库 App，而是「AI 个性化语料 + 点词精读 + 跟读打分 + 考官模拟」的练习闭环，外层包一层多邻国式的成长游戏。
- **目标用户**：以**冲 6.5 的大多数考生**为默认服务对象（市场最大、最需要「开口素材 + 即时反馈」）。目标分可在设置里切换（MVP 默认 6.5，支持其他分段）。
- **风格基调**：明亮、清新、年轻。配色走「薄荷奶白 + 珊瑚橙点睛」（方案 A）。

---

## 2. 核心练习闭环（最重要）

每张卡片 = 一道雅思真题。单张卡片的完整流程：

1. **出题**：App 提供真实雅思问题（按 Part 1 / 2 / 3 分类，可切换）。
2. **默认语料**：每张卡**默认就有一段 AI 生成的高分范本回答**，开箱即用（不强制用户先输入提示词）。
3. **个性化（可选）**：用户若想要专属回答，可输入提示词（"我喜欢爬山和摄影…"）→ AI 据此重新生成个性化语料。
4. **点词精读**：语料中的**雅思高频词/搭配**高亮可点（建议 AI 自动挑 4-6 个重点高亮，其余词长按也可查）→ 点击弹出音标、释义、搭配注释。
5. **示范朗读**：语料可整段 TTS 示范朗读（真人感发音）。
6. **跟读练习（阶段一）**：见下「两阶段练习」。
7. **模拟作答（阶段二）**：见下「两阶段练习」。
8. **消除**：语音达标 → 卡片消除（点亮关卡进度）。

### 设计原则
- **零摩擦先动起来**：默认语料直接可用，降低开口前门槛。
- **反馈少而狠**：每轮只给「如果只改一件事，改这个」的单点建议，避免一次列十几个错误劝退用户。

---

## 3. 两阶段练习

一张卡片的「语音输入」分两个阶段，顶部 segment 切换：

### 阶段一 · 跟读练习
- 长按录音，跟读语料。
- AI 打分（**仅评发音 + 流利度两项**，标准写死，每次给分附理由）。
- 可**反复练习**，不限次数。
- 提供「**隐藏回答 · 背诵模式**」开关：隐藏语料后凭记忆背诵。
- **达标线比目标分低 0.5**（目标 6.5 → 跟读到 6.0 即放行消除），把严格标准留给考官模式，避免游戏化爽感被打分挫败抵消。

### 阶段二 · 模拟对话
- 隐藏语料，凭自己的话作答。
- **不要求逐词一致，意思大致一致即可**评分通过。

> **MVP 取舍**：把「默认语料 + 点词 + 跟读打分」这条线打磨到顺（用户每天碰最多次），模拟对话与考官 Boss 做成「能用但简化」的版本，集中调教精力在跟读环节。

---

## 4. 闯关系统 · 登山地图（主页）

### 隐喻
**爬山**：从左下角山脚出发，沿盘山小路一路往右上角攀登，登顶 = 达到该组目标。海拔上升 = 备考进度。

### 结构（关键）
- **三大章节 = Part 1 / 2 / 3**，顶部用**大标题切换**（当前 Part 放大加粗，其余缩小灰显）。每个 Part 独立。
- **一组话题 = 一屏 = 一座山峰**：每座峰含 **3 个话题关 + 峰顶 1 个小 Boss**。
- **每个 Part 含约 3 座峰（3 组）**，顶部标「第 X / 3 峰」。
- **节点 = 小圆点**，精准贴在盘山路（虚线小径）上，像路上的一个个站点。
- **峰顶 = 小 Boss**（该组的考官模拟），比普通关稍大、可加小旗，突出「这座峰的终点」。
- **组间切换**：登顶打完小 Boss → **纵向上滑丝滑切换**到更高的下一座山峰（一峰更比一峰高）。当前屏不偷看下一座峰，保持干净。

### 解锁规则
- **严格线性**：必须按顺序逐关完成，不能跳关。
- 当前关：橙色高亮 + 光晕 + 小人停驻。
- 已通关：绿色小圆 + 勾（走过的「脚印」）。
- 未解锁：淡灰小圆（**靠暗淡表示锁定，不用锁图标**）。

### 交通工具（成长感，MVP 做装饰性）
- 玩家小人乘交通工具沿路前进。
- **每组换一档**（第一组步行/火车 → 过该组小 Boss 后升级为更高级交通工具，如飞机）。升级踩在「通过小 Boss」的奖励时刻。
- MVP 阶段交通工具仅为装饰性升级，不绑定能力指标。

---

## 5. 考官模式（小 Boss / Boss 模拟）

- **触发**：每组 3 个话题关全部消除后，解锁该组峰顶的小 Boss（考官模拟考）。
- **意义**：真模拟。攒够一组话题素材后，来一次连续的考官追问实战，让 Boss 有仪式感和模拟价值，而不是廉价的逐关小测。
- **形态**：AI 扮演考官，按该 Part 的真实问答节奏提问 + 追问；用户语音作答。
- **评分**：**意思达标即过，不要求逐词一致**；给 band 分（如 6.0 / 6.5）+ 单点改进建议。

---

## 6. 复习机制 · 今日复习补给

- 不重回原关卡（太繁琐）。
- 结合**遗忘曲线 + 用户历史得分**，**每天动态生成一个独立的「复习小关卡」**，需点亮。
- 在地图上以**底部「今日复习补给」卡片**呈现（不挤占主路径节点）。
- 点亮可得经验奖励（如 +20 经验）。
- **目的**：语料和发音会忘，间隔重现保证考场能调取，确保实际提分效果。

---

## 7. AI 调用点与一致性要求

产品有 3 个 AI 判定环节，每个都需稳定输出：

| 环节 | 用途 | 关键要求 |
|---|---|---|
| 生成语料 | 默认范本 + 个性化回答 | 按目标分调难度；自动挑 4-6 个高频词高亮 |
| 跟读打分（阶段一） | 评发音 + 流利度 | **维度收窄、标准写死在 prompt、每次附理由**（维度越少一致性越高，用户越信任） |
| 考官模拟（小 Boss） | 追问 + 综合评分 | 意思达标即过；给 band 分 + 单点建议 |

> **信任地雷提醒**：AI 打分忽高忽低、同段话两次分不同，会瞬间摧毁用户信任。MVP 务必把打分维度收窄、标准固定。

---

## 8. 全局设定

- **目标分**：MVP 默认 6.5；设置页可切换其他分段，切换后影响语料难度、打分宽容度、词汇高亮权重（主要是改 system prompt 权重，不改产品结构）。
- **Part 切换**：Part 1 / 2 / 3 顶部大标题切换。
- **底部导航**：登山（主页）/ 练习 / 进度 / 我的。导航栏无白条，融入背景。

## 9. 视觉设计规范

### 配色（方案 A · 薄荷奶白 + 珊瑚橙点睛）
| 用途 | 色值 |
|---|---|
| 背景奶白 | `#FAFCFB` |
| 已通关 / 主绿 | `#3BC49A`（深一级 `#1D9E75` / 文字 `#0E7A5E`） |
| 当前关 / 点睛橙 | `#F0875A`（文字 `#C75A2C` / 光晕 `#FCE3D7`） |
| 山体远近层 | `#EAF3EF` / `#DCEDE6` |
| 路径 | 底 `#CFE4DB` + 白色虚线 `#FFFFFF` |
| 未解锁灰 | `#E2EEE8`（文字 `#8AA39A` / `#A9BDB5`） |
| 复习补给（橙系卡片） | 底 `#FCE3D7` 边 `#F0875A` |

### 形态
- 大圆角、扁平、留白充足，多邻国式明亮卡通。
- 关卡节点：小圆点，吸附在盘山路径曲线上。
- 未解锁状态用「暗淡」表达，不用锁图标。
- 横向滚动条隐藏。

### 待精修项（已知，留待后续）
- 小圆点严格吸附路径曲线（目前为手工摆放）。
- 小人 / 交通工具换成自绘角色（目前为占位图标）。
- 密集排布时节点文字标签的避让（或改为点击弹气泡）。
- 峰顶小 Boss 加小旗 / 稍大处理。
- 上滑切峰时的云雾 / 海拔渐变强化高度感。

## 11. 技术建议（vibecoding）

- **MVP Web 先行**验证玩法（与历史项目偏好一致：React Native + Expo 可作为移动端延展方向）。
- 语料生成 / 打分 / 考官模拟均调 LLM API；TTS 用于示范朗读；语音输入用 ASR 转写后送评分。
- 打分类 prompt 务必固定维度与评分标准，输出结构化 JSON（band 分 + 单点建议）便于 UI 渲染。

---

## 12. 已达成的关键产品共识（复盘留档）

1. **默认锚 6.5**，不锚 7（卡片背语料法最匹配中分段冲刺人群）。
2. **默认语料 + 可选个性化**，不强制用户先写提示词（零摩擦）。
3. **3 话题一组，组末小 Boss**（Boss 不廉价、有真模拟价值）。
4. **加入复习机制**（每日独立复习关，遗忘曲线 + 得分驱动）。
5. **打分一致性优先**：维度收窄、标准写死、每次附理由。
6. **MVP 聚焦跟读环节**，模拟 / 考官做简化版。

## 13. 卡片练习页设计稿（初稿）

### 页面目标

卡片练习页是用户每天停留最久的核心练习场。它不应该像考试系统，也不应该像长文章阅读器，而应该像一张可以被逐步吃透的「口语素材卡」：先看到题，再拿到默认答案，然后通过点词、听读、跟读、背诵和模拟作答完成消除。

### 信息架构

页面从上到下分为 5 个稳定区域：

1. **顶部状态栏**
   - 左侧：返回地图按钮。
   - 中间：Part 与当前话题组，例如 `Part 2 · Experiences`。
   - 右侧：目标分胶囊，例如 `Target 6.5`。
   - 顶部下方显示当前关卡进度，例如 `2 / 3 cards`，用细进度条表达，不抢主视觉。

2. **题目区**
   - 使用浅薄荷底的独立题目块。
   - Part 1 / 3：展示单个问题。
   - Part 2：展示 cue card 结构，包含 topic + 3-4 个 bullet prompts。
   - 题目区右上角可放一个小标签：`Current Card` / `Review` / `Boss Prep`。

3. **语料卡区**
   - 默认展开 AI 生成范本回答。
   - 回答文本采用大行距、短段落，不做密集长文。
   - 自动高亮 4-6 个重点词 / 搭配，颜色使用珊瑚橙下划线或浅橙底。
   - 点击高亮词弹出单词详情弹层；长按任意词也可查词。
   - 右上角提供三个轻量工具：
     - `生成个性化`：打开提示词输入抽屉。
     - `隐藏回答`：进入背诵模式。
     - `示范朗读`：播放 TTS。

4. **练习控制区**
   - 顶部为 segment：`跟读练习` / `模拟作答`。
   - 默认停在跟读练习。
   - 中间是大号录音按钮，长按录音，松手提交。
   - 录音按钮周围显示录音波形或呼吸光效。
   - 按钮下方显示当前目标：
     - 跟读练习：`跟读到 6.0 即可通关`
     - 模拟作答：`意思大致完整即可`

5. **反馈与通关区**
   - 未练习时显示一句温和提示：`先跟读一遍，系统只看发音和流利度。`
   - 提交后显示：
     - band 分，例如 `6.0`
     - 两个小维度：`Pronunciation`、`Fluency`
     - 单点建议：`如果只改一件事：把句尾 /t/ 收清楚。`
   - 达标后底部主按钮变为 `点亮这一关`，点击后回到登山地图并播放节点点亮反馈。

### 跟读练习状态

#### 初始态
- 语料完整可见。
- 示范朗读按钮突出但不强制用户先听。
- 录音按钮文案：`按住跟读`。
- 底部提示强调低压力：`可以反复练，不限次数。`

#### 录音中
- 录音按钮变为珊瑚橙。
- 显示实时计时，例如 `00:18`。
- 语料卡可跟随朗读进度轻微高亮当前句（MVP 可先不做逐词同步，只做整句手动高亮或不高亮）。

#### 评分中
- 按钮进入 loading。
- 文案：`正在听你的发音...`
- 保持页面结构不跳动，避免等待时焦虑。

#### 未达标
- 显示 band 分与单点建议。
- 主按钮保持 `再练一次`。
- 次按钮：`听示范`。
- 不展示长错误列表。

#### 已达标
- 显示 `已达到本关通过线`。
- 主按钮：`点亮这一关`。
- 次按钮：`再练一次`，允许用户刷更高分但不强迫。

### 模拟作答状态

模拟作答在 MVP 中做简化，但必须保留真实考试感：

- 默认隐藏语料，只展示题目。
- 用户用自己的话回答，不要求背诵一致。
- 评分只判断意思是否覆盖核心点，并给 band + 单点建议。
- 若通过，显示 `模拟通过`；若未通过，建议只指出一个内容缺口。
- 模拟作答不作为普通卡片消除的强制条件，优先保持主线顺滑。

### 个性化语料抽屉

从语料卡右上角点击 `生成个性化` 打开底部抽屉：

- 输入框标题：`加入你的真实经历`
- 占位示例：`比如：我喜欢爬山和摄影，上次旅行去了云南。`
- 按钮：`重新生成回答`
- 生成后替换当前语料，并保留原问题与高亮词逻辑。
- 提供 `恢复默认语料` 入口，避免用户生成后后悔。

### 单词详情弹层

点击高亮词后，底部弹层展示：

- 单词 / 搭配本身。
- 音标与播放按钮。
- 中文释义。
- 这句话里的用法解释。
- 1 个可迁移表达，例如 `come in handy = 派上用场`。
- 按钮：`加入生词本`。

弹层不遮住整张语料卡，保留上方上下文，让用户知道这个词来自哪句话。

### 视觉规则

- 页面背景沿用奶白 `#FAFCFB`。
- 题目区用浅薄荷底，语料高亮用珊瑚橙。
- 录音按钮是页面唯一大面积珊瑚橙元素，保证主操作明确。
- 卡片圆角可以略大，但不要做多层卡套卡；题目、语料、反馈区应像同一张练习面板里的分区。
- 文字排版要短行、松行距，避免雅思语料看起来像背诵惩罚。

### MVP 取舍

第一版必须做：
- 题目展示。
- 默认语料展示。
- 4-6 个重点词高亮。
- 单词详情弹层。
- TTS 示范朗读入口。
- 跟读录音。
- 发音 + 流利度评分。
- 单点建议。
- 达标后点亮关卡。

第一版可以简化：
- 个性化语料可以先用单输入框，不做复杂偏好表单。
- 当前句同步高亮可以先不做。
- 模拟作答可以先走同一套录音与反馈组件。
- 生词本可以先只做本地收藏状态，不急着完整管理页。
